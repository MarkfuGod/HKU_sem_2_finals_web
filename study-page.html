03 / Core Concepts · Visualized

核心概念 — 10 张图把这门课看懂

每个概念都遵循三段式：① 是什么（人话）→ ② 一张图 → ③ 考试怎么用。看完这 10 个，这门课的骨架就稳了。

① 欺诈三角 — Fraud Triangle · "为什么有人会作案？"

想象一个普通员工小李。他 不会无缘无故去贪钱。但当下面三件事 同时 发生时，他就有可能：

  • ① Pressure（压力 / 动机）：小李欠了赌债 / 想赶 KPI / 老婆住院。
  • ② Opportunity（机会）：财务系统漏洞，他报多少都没人查。（这是唯一一个公司可以控制的因素 → 内控的目标就是堵这一条）
  • ③ Rationalization（合理化）：他对自己说："公司这么有钱，我就拿一点应急，下个月就还。"
!

顺序按教材：Pressure → Opportunity → Rationalization。不要写成 Opportunity 在前，老师会扣分。

★

The Three R's（坏人特征 — 必背 4 条 / 2025 May Q1.1）：
Rum（酗酒 / 染瘾）· Racecars（飙车 / 挥霍 / 生活方式远超收入）· Romancers（风流 / 自负） + 控制欲强 / 拒绝休假 / 生活骤变。

② 红旗 + 假阳 / 假阴 — Red Flag, FP vs FN · "抓人会冤枉好人吗？"

红旗预警（Red Flag）就是"这地方有鬼，需要查"的信号——它不是定罪证据，只是告诉你"喂，这里不对劲"。常见的红旗：

  • Outliers（极端离群值）：报销 1 万和报销 100 万。
  • Inliers（值在正常范围但本不该出现）：发票里出现一个不存在的部门代码。
  • Missing Sequences：发票号 1001、1002、1004 — 中间的 1003 哪去了？
  • Duplicates：同一供应商、同一金额被付了两次。
  • Round Numbers：所有报销都是整数 $1000，没有 $998.43。
  • Unexpected Timing：周日凌晨 3 点提交报销。

那"抓人"会出哪两种错？

  • FP — 假阳性 / 误报：好人被你抓了。
    → 客户被冻结、投诉、流失。
  • FN — 假阴性 / 漏报：坏人没抓到。
    → 公司被骗、监管罚款、上头条。
⚑

考试核心：Recall 防漏报，Precision 防误报。 哪个更重要 = 哪种错的成本更高。

③ 洗钱三阶段 — Money Laundering 3 Stages · "黑钱怎么变白？"

把洗钱想成"把脏衣服洗干净"：先扔进洗衣机（放置）→ 高速搅拌让它面目全非（离析）→ 烘干后挂起来当新衣服穿（融合）。

i

常考题型：①写出三个阶段（5 分送分）②举一个例子（如毒贩把现金 → 拆成小额存银行 → 转给海外空壳公司 → 买香港楼）。
三阶段顺序千万别写反：放、离、融。

⑤ 不平衡数据 + SMOTE — Class Imbalance · "1% vs 99% 怎么办？"

问题：真实欺诈数据，欺诈往往只有 1%，正常占 99%。一个偷懒的模型只要"全猜正常"，准确率就 99% —— 但一个欺诈也没抓到。这就是"Accuracy 陷阱"。

3 种主流解法：

  • Over-sampling 过采样：把欺诈样本复制几倍。
    ⚠️ 风险：模型把同一个样本"背"下来 → 过拟合。
  • Under-sampling 欠采样：从正常样本里随机扔掉一大半。
    ⚠️ 风险：丢掉了大量信息。
  • SMOTE（合成少数类过采样）：
    找两个真实欺诈样本，在它俩之间画一条线，线上挑新点当合成欺诈。
    优点：不重复、不丢信息。
    ⚠️ 缺点：如果欺诈样本里混进了一个"长得像正常"的离群点，SMOTE 会在两类之间架起"桥梁"，模糊决策边界。
★

考试万能话术：当题目说"数据极度不平衡"，先答 "Accuracy 是误导指标"，然后推荐 Recall / F1 / PR-AUC，最后提一句 SMOTE / Cost-sensitive learning / Class Weight。这 3 段就能拿满。

⑥ 本福特定律 — Benford's Law · "数字会自己暴露作假"

魔法事实：自然产生的数字（发票金额、河流长度、人口数）的首位数字不是均匀分布的。1 出现的概率约 30.1%，9 只有 4.6% —— 1 几乎是 9 的 7 倍。

为什么人造数据会暴露？因为人编数据喜欢编"看着随机"的数字，比如尽量平均地用 1–9，或者特别喜欢 5、6（为了刚好低于审批门槛 $1000 → 写 $999、$995）。

怎么用？卡方检验（Chi-square goodness-of-fit）：

公式χ² = Σ (Oi − Ei)² / Ei    df = 8    临界值 = 15.51
  • Oi = 实际观察到的首位是 i 的次数
  • Ei = 按 Benford 公式 log₁₀(1 + 1/d) × n 算出的期望次数
  • 若 χ² > 15.51 → 拒绝原假设 → 数据偏离 Benford → 红旗！
!

三大限制（必背）：① 不符合不等于"有罪"，只是红旗；② 知道这个定律的犯罪分子可以伪造符合的数据；③ 不适用于被指派编号（如 ID 号）和受约束数字（如固定价格 $9.99）。

⑦ 决策树 + Gini — Decision Tree, Gini Impurity · "怎么把人切成两堆？"

决策树就是一连串的"是 / 否"问题。比如："信用分 < 700？" → 是的去左边，不是的去右边。怎么知道哪个问题问得最好？

看哪个问题问完之后，两堆人变得最"纯"（最纯就是一堆全是好人，一堆全是坏人）。"不纯"用 Gini 不纯度衡量：

GiniGini = 1 − Σ pi²  ·  越小越纯

举例：一堆 10 人，5 好 5 坏 → Gini = 1 − (0.5² + 0.5²) = 0.5（最不纯，乱糟糟）
一堆 10 人，10 好 0 坏 → Gini = 1 − (1² + 0²) = 0（完美纯）

分裂之后看 加权平均的 Gini，下降越多说明这个分裂越好。

ΔGiniΔGini = Gini(父) − [w₁·Gini(左) + w₂·Gini(右)]
i

另一个常考公式：信息增益（Entropy 版） — 网络分析卷 Q1.1 考过：

Entropy = −Σ pi·log₂(pi)
IG = Entropy(父) − Σ w·Entropy(子)

选 IG 最大（或 ΔGini 最大）的特征做根节点。

📐 类别变量距离度量 — SMC vs Jaccard · 抓骗子的"相似度"该怎么算？

故事场景：你设计了 5 个"红旗规则"标记可疑理赔单（30 天内出险、金额 > $10,000、出险地是欺诈热点……）。每张理赔单经过规则后得到一串 Yes/No。问题来了：两张理赔单到底有多像？ 这道题答错，整个 K-Means 聚类就废了。

① 先把对比拆成 2×2 表（理解一切的钥匙）

B = YesB = No
A = Yesa（俩都触发 = Yes-Yes 匹配）b（A 触发 B 没）
A = Noc（B 触发 A 没）d（俩都没触发 = No-No 匹配）

总数 = a + b + c + d；意见一致 = a + d；不一致 = b + c。整个 SMC vs Jaccard 之争只围绕一件事：要不要把 d（No-No）算进相似度里？

② 两个公式拆开看

SMC(a + d) / (a + b + c + d)

→ 所有意见一致的（无论"都说有"还是"都说没"）÷ 总数。

Jaccarda / (a + b + c)

→ 俩都说有的 ÷（除"俩都没"以外的所有情况）。把 d 从分子分母全删掉。

③ 用具体数字算给你看（场景递进，看出差距）

场景 A · 5 个红旗 · 俩客户都触发了红旗 1、3

A = [Y, N, Y, N, N]，B = [Y, N, Y, N, N] → a=2, b=0, c=0, d=3

  • SMC = (2+3)/5 = 1.0
  • Jaccard = 2/(2+0+0) = 1.0

两个都是 1.0 —— 看似一样，别急，看下一个。

场景 B · A 触发 1、3，C 啥红旗都没触发

A = [Y, N, Y, N, N]（可疑），C = [N, N, N, N, N]（完全干净） → a=0, b=2, c=0, d=3

  • SMC = (0+3)/5 = 0.6 ← 居然有 60% 相似度！
  • Jaccard = 0/(0+2+0) = 0.0 ← 零相似度，正确！

差距出现了。 A 是有 2 个红旗的可疑客户，C 是完全干净的客户，本不该像。但 SMC 把"俩都没触发红旗 2、4、5"也算成"很像"，给了 0.6 的虚高分；Jaccard 不在乎"都没触发"，给了 0 分（正确）。

场景 C · 100 个红旗 · 平均触发 2 个（PDF p.21 真题原场景）

A 触发红旗 1、3；C 触发红旗 50、77；其余 96 个俩人都 No → a=0, b=2, c=2, d=96

  • SMC = (0+96)/100 = 0.96 ← 96% 相似！
  • Jaccard = 0/(0+2+2) = 0.0 ← 0% 相似

这就是灾难。 用 SMC 的话，公司里所有客户相似度都 ≥ 0.95，K-Means 跑出来的所有人都被分到一个簇 —— 因为大家共同点都是"我们都没触发那 98 个稀有红旗"。Jaccard 一刀砍掉这种"虚假相似"，立刻能分清谁跟谁真的像。

⑥ A+ 满分话术（PDF L8 p.21 送分题模板）

★

题目：「如果有 100 个红旗指标，平均只触发几个，应该用 SMC 还是 Jaccard？」

应该用 Jaccard 指数。在这种红旗稀疏触发 (sparse) 的场景下，绝大多数指标对任意两个客户而言都是 No-No 匹配（俩人都没触发）。如果用 SMC，这些 No-No 匹配会被算进分子，导致所有客户的相似度都被人为推高（接近 1），完全无法区分谁可疑谁正常，K-Means 聚类会失效。Jaccard 把 No-No 匹配从分子分母里都剔除，只衡量"在至少有一方触发了的红旗里，俩人有多少是共同触发的"，从而真正捕捉"两个可疑客户是不是被同样的红旗组合识别出来的"，有效用于聚类和异常检测。

⑧ K-Means + 肘部法则 — Clustering, Elbow Method · "无标签时怎么分组？"

K-Means 是干嘛的？ 当你没有标签（不知道谁是欺诈），但想看有没有抱团的可疑群体（比如医保欺诈团伙），就用 K-Means 把数据点按"长得像"分成 K 个簇。

4 步算法：

  1. 随机选 K 个中心点
  2. 每个数据点归到最近的中心
  3. 重新算每个簇的平均值，作为新的中心
  4. 重复 2-3 直到中心不再动

K 怎么选？肘部法则：

  • 横轴：K 取 1, 2, 3, ...
  • 纵轴：WSS（簇内误差平方和，越小越紧密）
  • WSS 一开始下降很快，到某个 K 之后下降明显放缓 → 这个拐点就是"肘部 (Elbow)"。
★

2025 Dec Q1.3 考点：给 WSS 表，要求选 K + 解释为什么某个簇可疑。
可疑簇模板："这个簇虽小（占比 0.06%），但同时满足 赔付高 + 住院短 + 距离短——典型医患合谋骗保特征 → 触发调查。"

⑨ 孤立森林 — Isolation Forest · "异常值更容易被一刀切出来"

核心直觉："异常值是少数且不同的 (few and different)"。所以如果你随便切几刀把数据切开，异常值会很快被孤立出来，而正常点要切很多刀。

  • 对每个数据点，记录它被孤立时切了几刀 = Path Length（路径长度）
  • 路径越短 → 越容易被切出来 → 越异常
  • 多棵树平均路径长度 → 算 Anomaly Score

优点：

  • 不用算距离，速度极快 O(n log n)
  • 适合大规模高维金融数据
  • 不需要标签（无监督）
vs

iForest vs One-Class SVM vs K-Means：

  • iForest 抓"孤狼"（极端异常）
  • One-Class SVM 学"正常的边界"，落在外面就异常
  • K-Means 找"团伙"（一群相似可疑账户）

⑩ 社交网络分析 — Social Network Analysis · "看关系网抓团伙"

核心思想：连坐 (Guilt-by-association)。如果你的邻居一堆是欺诈犯，你大概率也有问题。同质性 (Homophily)："物以类聚" —— 人喜欢和相似的人建联系。

3 大类网络指标：

① 邻居指标（直接邻居）：

  • Degree（度）：连了几个人
  • Triangles（三角形）：3 个人两两相连 = 紧密小团伙 = Fraud Ring
  • Density（密度）：实际边数 / 最大可能边数 ＝ 2M / N(N-1)
  • Relational Neighbor：邻居中欺诈比例

② 中心性指标（在网络中的"地位"）：

  • Closeness：到所有人的平均距离的倒数 → 高 = 消息传得最快
  • Betweenness：处于多少条最短路径上 → 高 = 桥梁/中介（典型例子：钱骡 money mule）

③ 集体推断（PageRank）：

  • 把欺诈节点当"种子"，让分数沿网络扩散
  • 未标记节点的 PageRank 分数高 → 受欺诈影响最大 → 重点监控
!

网络指标只是风险暴露（risk exposure），不是定罪证据。Degree 高也可能只是因为他人脉广（CEO），要结合"邻居质量"看。

② 单变量分析 — 先看每一列长什么样

Univariate Analysis（单变量分析） = 一次只看一列数据。就像你刚到一个公司上班，先一个一个同事认识，而不是先研究办公室政治。

Centrality（集中趋势）

数据「中心」在哪？

  • Mean（均值）：所有值加起来除以个数。
  • Median（中位数）：排序后中间那个。对 outlier 不敏感。
  • Mode（众数）：出现次数最多的。用于 categorical 类别变量（连续变量没意义）。

Dispersion（离散程度）

数据散不散？

  • Range：最大减最小。
  • Variance / Standard Deviation：偏离均值的平均距离。
  • Skewness（偏度）：分布是不是左右对称？右偏（正偏）= 长尾在右边。
  • Kurtosis（峰度）：分布是尖头还是平头？
算一遍 · 5 个交易金额 [10, 12, 15, 18, 1000]
  • Mean = (10+12+15+18+1000) / 5 = 211（被 1000 拉爆了）
  • Median = 15（不被 outlier 影响）
  • Skewness：明显右偏（一个超大值在右边）
!

必考结论：当数据有严重 outliers 时，用 Median 比 Mean 更可靠。这就是为什么课件说「金融交易金额几乎都是 right-skewed（右偏）」。

可视化武器（每个都可能考）

图表干啥用的一句话记
Histogram（直方图）看单个变量的分布「这个数集中在哪？」
Box Plot（箱线图）看中位数 + outlier「有没有异常值？」
Scatter Plot（散点图）看两个变量的关系「X 和 Y 一起涨吗？」
Pie Chart（饼图）看类别比例「几种里面哪种多？」
Heatmap（热力图）看相关系数矩阵「哪两列最铁？」

③ 相关性分析 — 看变量之间的「关系」

Correlation Analysis（相关性分析） = 看 X 涨的时候，Y 跟着涨还是跌。Pearson Correlation（皮尔逊相关系数） 是最常用的：

Pearson rr = Cov(X, Y) / [ σX · σY ]

人话：两个变量一起变的程度（协方差）÷ 它们各自变化的范围。

数值含义（必背阈值）

r 值含义

±0.3 ~ ±0.5中等相关
< ±0.29Low correlation（低相关）
0完全没关系

假如算出 Correlation(信用卡交易金额, 用户年龄) = -0.8：

  • 翻译：「年龄越大的人，交易金额越小」
  • 用途：「那我们可以把这两个变量都喂给模型，因为它们对欺诈识别都有信息量。」

④ 信用卡欺诈案例的三大发现（必背三句话）

课件用真实的 Kaggle 信用卡欺诈数据跑了一遍 EDA，得出 3 个关键 Finding：

  1. Finding 1（单变量）：交易金额严重右偏（highly skewed），多数是小额，少数是巨额 outlier。
  2. Finding 2（多变量）：欺诈交易的金额平均比正常的小！（反常识 —— 骗子怕被发现，故意小金额多次刷。）
  3. Finding 3（核心）：数据严重不平衡（severely imbalanced）—— 99% 是正常交易，1% 是欺诈。
!

必考考点：考试可能会问「反直觉的发现是什么？」—— 答 Finding 2。

⑤ 特征工程 — 把数据「加工」成模型爱吃的样子

A. 归一化 / 标准化（Normalization）

为什么要做？不同变量数量级差太多，模型会「偏心」大数。

例子：年龄 [0~100]，收入 [10000~100000]。模型一看，收入数字大，影响力大 100 倍 —— 这是错觉，不是真相！

两个必考方法：

Min-Maxx' = (x − min) / (max − min)

全部压缩到 [0, 1]

Z-scorex' = (x − μ) / σ

转成均值 0、标准差 1

Min-Max 算一遍

数据 [10, 20, 30, 40, 50]，min = 10, max = 50
30 归一化后 = (30 − 10) / (50 − 10) = 0.5

Z-score 算一遍

数据均值 μ = 30, σ = 15
30 标准化后 = (30 − 30) / 15 = 0

i

送分题套路：题目给你 5–6 个数，问你「用 Min-Max 归一化」或「算 Z-score」，套公式就完事。

B. 离散化（Discretization / Binning）

这是什么？把连续变量切成几段，变成类别变量。

  • AGE 从数字 25, 38, 60 → 变成类别 youth, adult, elderly
  • 把出现次数少的类别（< 5%）合并成 OTHER

两种必考分箱方法：

方法怎么切例子（数据 1000~2000）
Equal Interval（等距）按值的范围切[1000–1500]、[1500–2000]
Equal Frequency（等频）按数量切，每箱人数一样每箱各 3 个客户
算一遍 · 客户收入 A=1000, B=1200, C=1300, D=2000, E=1800, F=1400，分 2 箱
  • 等距分箱（范围 1000~1500 和 1500~2000）：BIN1 = {A, B, C, F}，BIN2 = {D, E}
  • 等频分箱（每箱 3 个）：排序后 = 1000, 1200, 1300, 1400, 1800, 2000 → BIN1 = {A, B, C}，BIN2 = {F, E, D}
!

重要警告：这两种方法都没考虑目标变量（target，比如 fraud） —— 只是机械切箱。

⑥ 降维 — 别让模型被一堆没用的变量淹死

Dimensionality Reduction（降维） = 数据有 100 列，挑出最重要的 10 列，扔掉剩下 90 列。

为什么要做？

  • 效率高：训练快、内存少。
  • 可解释：少几个变量，画图能看懂。
  • 去噪：欺诈检测里平均只有 10–15 个变量真正有用。

两条路线（必区分）

方法做法类比
Variable Selection（变量选择）直接扔掉没用的列「你被开除了，回家。」
Variable Extraction（变量提取）把多列揉成新的少数列「你们三个合并成一个部门。」
i

PCA 属于哪种？Variable Extraction —— 它把原始变量做线性组合搞出「主成分」，不是直接砍。

PCA（主成分分析）—— A+ 必懂

Principal Component Analysis（主成分分析） 的核心思想：找一条新坐标轴，让数据投影上去方差最大（信息保留最多）。

比喻：你拍一个茶壶的影子。

  • 从正上方拍 → 看到一个圆 → 信息少（方差小）
  • 从侧面拍 → 看到茶壶的轮廓 → 信息多（方差大）

PCA 就是自动帮你找「侧面」那个最佳角度。

必背公式：

PCAZ = Y · V
符号含义
Z主成分得分矩阵（n × m）
Y标准化后的原始数据（n × p）
V特征向量矩阵（p × m）
n样本数
p原始变量数
m保留的主成分数
课件里的关键例子 · 3 个变量 → 2 个主成分
  • PC1 解释了 79% 的方差
  • PC2 解释了 15% 的方差
  • PC1 + PC2 = 94% → 几乎不丢信息，但变量从 3 降到 2

结论：50 多个变量的数据，用前几个 PC 就够了。

?

常考短答：「为什么 PCA 选的是方差最大的方向？」
答：「方差代表信息量。方差越大，区分度越强，丢失的信息越少。」

⑦ 类别不平衡 — 欺诈检测的头号敌人

Imbalanced Dataset（不平衡数据集） = 一类样本远多于另一类。典型欺诈数据：99% 正常，1% 或更少是欺诈。

为什么是大问题？（必背三句话）

  1. 大多数 ML 模型默认类别均衡。
  2. 模型会偏爱多数类（光猜「正常」就 99% 准确率！）。
  3. 导致模型完全找不到欺诈 —— 这正是我们要的东西！
!

重灾区考点：考试问「为什么不能直接用 Accuracy 评估欺诈模型？」—— 答案就在这里：因为不平衡数据下 Accuracy 没意义。

五个解决方案（必背全套）

方法做法优点缺点
Over-sampling（过采样）复制少数类不丢信息容易过拟合（模型背了答案）
Under-sampling（欠采样）删多数类训练快丢失信息
ROSEOver + Under 混合 + 生成数据平衡两者调参复杂
SMOTE给少数类生成合成样本不复制、不丢失异常值会「搭桥」到多数类
ADASYNSMOTE 进阶版，关注边界样本自动适应密度实现略复杂

SMOTE 算法步骤（必考！）

Synthetic Minority Over-sampling Technique 4 步走：

  1. 从少数类（欺诈）中随机选 1 个观察值。
  2. 找它的 5 个最近邻（K-Nearest Neighbors，都在少数类里）。
  3. 从 5 个邻居里随机选 2 个。
  4. 在选中的邻居和原点之间，随机生成 2 个新点。
!

SMOTE 的问题（必考）：如果某个少数类样本是 outlier 又混进了多数类区域，SMOTE 会在它和邻居之间画线 → 造出「桥」连到多数类，反而搞乱模型。

ADASYN 的进阶逻辑（计算题）

Adaptive Synthetic Sampling 的核心：越靠近多数类边界的少数类样本，越要给它造合成样本。

Impurity RatioIR = (多数类邻居数) / K

课件原表（必看）：

Obs2411/5 = 0.2

结论：Obs3 的 ratio 最高（0.8）→ 给它造最多的合成样本（是 Obs2 的 4 倍，因为 0.8 / 0.2 = 4）。

A+ 话术：「ADASYN 比 SMOTE 智能，它把资源花在『危险地带』的样本上。」

⑧ 数据分割 — 别让模型考前看答案

三个必区分的数据集（必考概念题）

数据集干嘛用类比

Validation（验证集）调超参数（如决策树停止生长）课堂随堂测（边学边调）
Testing（测试集）最终评估期末考试（独立判分）

两种切分方案

场景切法
不需要 Validation70% 训练 + 30% 测试
需要 Validation40% 训练 + 30% 验证 + 30% 测试

为什么需要 Validation？（必考简答题）

Validation set 用来在训练过程中调整超参数（hyperparameter tuning）。如果只用 Training 来调，模型会过拟合训练数据；如果用 Testing 来调，等于「考前看答案」，最终评估失去意义。所以 Validation 是个独立的「练手数据」，专门用来选最佳模型配置。最后再用从未碰过的 Testing 数据公正打分。

必考概念：Overfitting（过拟合）

Overfitting（过拟合） = 模型在 Training 上表现极好，但在 Testing 上拉胯。

比喻：学生死记硬背了所有练习题答案，一遇到没见过的新题就完蛋。

⑨ 完整数据预处理流程（万能模板）

记住这 5 步顺序，简答题直接套：

  1. Data Cleaning（数据清洗）：处理缺失值和异常值。
  2. Data Partitioning（数据分割）：切成 Training / Validation / Testing。
  3. Handling Imbalance（处理不平衡）：SMOTE / Oversampling。
  4. Feature Engineering（特征工程）：创造新变量。
  5. Feature Scaling（特征缩放）：标准化 / 归一化。
!

关键顺序点：先 Partition 再 Balance！为什么？因为 SMOTE 只能在 Training 上做，不能污染 Testing —— 否则你就是「考前给学生发答案」。

"Among the four candidate models, Neural Network achieves the highest Recall (0.900) and the highest F1-score (0.654). Given that the cost of missing a fraudulent transaction (FN) outweighs the cost of investigating a false alarm in this credit-card scenario, I recommend deploying Neural Network. However, if the bank also values explainability for regulatory reporting, Logistic Regression would be a safer choice despite slightly lower performance."

② Gini / 信息增益 → 选根节点 高频

题目模式：给一张数据表 + 2 个候选分裂条件（如"信用分 < 700" 和 "负债比 < 0.3"），问哪个该当根节点。

  1. 算父节点 Gini（或 Entropy）。
    数总样本里 Yes / No 各几个 → 求 pYes, pNo → Gini = 1 − pYes² − pNo²
  2. 对每个候选特征，分别按条件切两堆。
    左堆（满足条件）：数 Yes / No → 算 Gini左
    右堆（不满足）：数 Yes / No → 算 Gini右
  3. 算加权 Gini。
    Gini加权 = (n左/n) × Gini左 + (n右/n) × Gini右
  4. 算 ΔGini = Gini(父) − Gini加权。
    对两个特征都算一遍。
  5. 选 ΔGini 最大的当根节点。
    解释：因为它"切完之后两堆变得最纯"，分类效果最好。
·worked example · 取自 2024 Dec Q1.3 (简化为前 6 行)

6 人样本，2 人 Yes（违约）、4 人 No。

父 Gini = 1 − (2/6)² − (4/6)² = 1 − 0.111 − 0.444 = 0.444

特征 1：信用分 < 700
左堆（< 700, 2 人）：2 Yes / 0 No → Gini = 1 − 1² = 0
右堆（≥ 700, 4 人）：0 Yes / 4 No → Gini = 1 − 1² = 0
Gini加权 = (2/6)(0) + (4/6)(0) = 0
ΔGini₁ = 0.444 − 0 = 0.444

特征 2：负债比 < 0.3
左堆（< 0.3, 3 人）：0 Yes / 3 No → Gini = 0
右堆（≥ 0.3, 3 人）：2 Yes / 1 No → Gini = 1 − (2/3)² − (1/3)² = 0.444
Gini加权 = (3/6)(0) + (3/6)(0.444) = 0.222
ΔGini₂ = 0.444 − 0.222 = 0.222

结论：ΔGini₁ (0.444) > ΔGini₂ (0.222) → 选"信用分 < 700"当根节点。

!

常见踩坑：① 算 p² 时忘记平方；② 忘了乘"子节点权重"（n子/n父）；③ 题目要求 Information Gain (Entropy) 而你写了 Gini → 看清题目用的是哪个！

③ Logistic 系数 → 概率预测 2022 考过

题目模式：给 3 个候选模型的 R² / Adjusted R² + 系数表 → 选最优模型 → 代入新样本预测概率。

  1. 选模型：先看 Adjusted R²（不是普通 R²）。
    因为 Adjusted R² 惩罚了无用变量。挑 Adjusted R² 最大的那个模型。
  2. 写出 Logit 公式。
    Logit(P) = β₀ + β₁X₁ + β₂X₂ + ... + βnXn
    把题目给的系数代进去（注意正负号）。
  3. 代入新样本算 Logit。
    把新样本的特征值 X₁, X₂, ... 代进公式。
  4. 用 Sigmoid 反算概率。
    P = 1 / (1 + e−Logit)
    用计算器算 e 的负 Logit 次方。
  5. 与阈值（一般 0.5）比较，> 0.5 → 判定 Fraud。
·worked example

假设 β₀=−4, β₁=0.05 (Amount), β₂=2.0 (Distance)；新样本 Amount=$80, Distance=2 miles。

Logit = −4 + 0.05×80 + 2.0×2 = −4 + 4 + 4 = 4
P = 1 / (1 + e−4) = 1 / (1 + 0.0183) = 0.982
0.982 > 0.5 → 预测 Fraud ✓

i

系数解读题（必背两句）：
① "β > 0 表示该变量增加，欺诈的 log-odds 上升，概率上升。"
② "exp(β) = odds ratio，X 增加 1 单位，欺诈几率乘以 exp(β) 倍。"

④ Benford + 卡方检验 2023, 2025 May 考过

题目模式：给一组金额 → 算 expected vs observed → 卡方 → 判断是否偏离。

  1. 提取首位数字 + 数频次。
    例如 $1,200 → 首位 1；$5,000 → 首位 5。
    统计 1～9 各出现几次，得 Oi（observed count）。
  2. 算 expected count。
    用 Ei = log₁₀(1 + 1/d) × n （n 是总笔数）。
    或者直接套速查百分比表：30.1% / 17.6% / 12.5% / 9.7% / 7.9% / 6.7% / 5.8% / 5.1% / 4.6%，再乘 n。
  3. 列表算每一项 (Oi − Ei)² / Ei。
    建议用表格写在答卷上：列 d / Oi / Ei / 差值平方 / 除以 Ei。
  4. 求和得 χ²。
  5. 对比临界值 15.51（df=8, p=0.05）：
    · χ² > 15.51 → 拒绝 H₀ → 数据偏离 Benford → 红旗
    · χ² < 15.51 → 不拒绝 H₀ → 数据符合 Benford

⑤ K-Means 肘部法则 + 可疑簇解读 2025 Dec 新考

题目模式：给 K 与 WSS 的数值表 → 选最佳 K → 解释为什么某个小簇可疑。

Relational Neighbor

·worked example · SNA 卷 — 节点 A 的指标

已知 A 到 B/C/D/E/F/G/H 的最短路径分别是 2, 1, 1, 1, 1, 1, 2（共 7 个节点对）。

  • Farness(A) = 2+1+1+1+1+1+2 = 9
  • Closeness(A) = (8−1)/9 = 7/9 = 0.778
  • 如果 A 的邻居中有 2 个是已知欺诈，A 共有 5 个邻居 → Relational Neighbor = 2/5 = 0.40

4 大业务解读题答法（背下来直接套）：

  • 谁获取信息最快？ → Closeness 最高 的节点。
  • 谁控制信息流（中介）？ → Betweenness 最高 的节点（典型：钱骡 money mule）。
  • 谁最容易"近朱者赤"？ → Relational Neighbor 中欺诈比例最高 的节点。
  • 谁最需要监控？ → PageRank (Fraud) 最高且未标记 的节点（受欺诈影响最深）。

④ Linear Regression — 线性回归 · 预测一个连续数字

D-1 · 是什么 + OLS

想预测一个连续的数字（如"这次保险欺诈金额是多少"），就画一条最佳拟合直线穿过数据点。

公式Y = β₀ + β₁X₁ + β₂X₂ + ... + βnXn + ε
  • Y = 想预测的目标（target / dependent）→ 例：欺诈金额
  • X₁, X₂ ... = 用来预测的特征（features / independent）→ 例：年龄、索赔金额、严重程度
  • β = 每个变量对 Y 的影响（系数 / 斜率）；β₀ = 截距（X 全为 0 时 Y 的均值）

怎么找最好的那条线？ OLS（Ordinary Least Squares，普通最小二乘） — 让所有点到这条线的距离平方和最小，也就是最小化 MSE。

D-2 · 5 大评估指标（每个都可能考）

MAE(1/n)·Σ|yi − ŷi|

最简单直观，但对离群值不敏感，不常用。

MSE(1/n)·Σ(yi − ŷi)²

误差平方 → 对离群值敏感。

RMSE√MSE

单位与 Y 一致，最常用。

R²1 − SSres / SStot

越高越好，"模型解释了多少 Y 的变化"。

Adjusted R²1 − [(1 − R²)·(n − 1) / (n − p − 1)]

n = 样本数, p = 特征数。多变量模型必看这个，惩罚无意义的复杂度。

·worked example · 算 R²

真实欺诈金额 [10, 20, 30, 40, 50]（均值 = 30），模型预测 [12, 18, 28, 42, 50]：

  • SSres = (10−12)² + (20−18)² + (30−28)² + (40−42)² + (50−50)² = 4+4+4+4+0 = 16
  • SStot = (10−30)² + (20−30)² + (30−30)² + (40−30)² + (50−30)² = 400+100+0+100+400 = 1000
  • R² = 1 − 16/1000 = 0.984 → 模型解释了 98.4% 的变化，非常好

D-3 · R² vs Adjusted R²（必考辨析 — 2022 Dec Q1.1 已考）

情况R²Adjusted R²
加一个无关变量一定上升或不变（虚高）大概率下降（被惩罚）
加一个有用变量上升上升
大小关系恒有 R² ≥ Adj R²；只有 1 个变量时两者相等
多变量模型选谁？必看 Adjusted R²

D-4 · AIC / BIC（小考点，常出在简答）

  • AIC（Akaike Information Criterion）：AIC = 2k − 2·ln(L̂)，惩罚复杂度，越低越好。
  • BIC（Bayesian Information Criterion）：BIC = k·ln(n) − 2·ln(L̂)，惩罚比 AIC 更狠，越低越好。
  • 记一句话：R² 高、AIC/BIC 低 = 好模型。

⑤ Logistic Regression — 逻辑回归 · 预测"是不是欺诈"

E-1 · 为什么需要它？

线性回归预测出来的 Y 可以是 −1000 也可以是 1000000，但你想预测"是 / 不是"（0/1）怎么办？→ 用 Logistic Regression 把输出压扁到 [0, 1] 之间，天生适合二分类。

E-2 · 三个必懂概念（用一个例子串起来）

情境：10 个保险理赔，2 个欺诈、8 个正常。

概念公式这个例子里的值
Probability (p)
概率		欺诈次数 / 总次数p = 2/10 = 0.2
Odds
几率（赔率）		p / (1 − p)0.2 / 0.8 = 0.25（"欺诈 vs 不欺诈" = 1:4）
Log-Odds / Logit
对数几率		ln[p / (1 − p)]ln(0.25) = −1.386
★

为什么要用 logit？ 因为 logit 的取值范围是 (−∞, +∞)，正好可以套线性回归的公式！

LogisticLogit(P) = ln[P / (1 − P)] = β₀ + β₁X₁ + β₂X₂ + ... + βnXn
反推 PP = 1 / (1 + e−Logit)  ·  (Sigmoid 函数)

E-3 · 系数 β 怎么解读？（必考！）

系数符号含义
β > 0X 增加 → 欺诈的 log-odds 上升 → 欺诈概率上升
β < 0X 增加 → 欺诈概率下降
β = 0X 和欺诈无关
exp(β) = Odds RatioX 每加 1 单位，欺诈几率乘以 exp(β) 倍
·worked example · 2022 Dec Q1.1 同款

假设 LR 模型：β₀ = −4, β₁ = 0.05 (Amount), β₂ = 2.0 (Distance)；新样本 Amount = $80, Distance = 2 miles。

Step 1：算 Logit = −4 + 0.05×80 + 2.0×2 = −4 + 4 + 4 = 4

Step 2：P = 1 / (1 + e−4) = 1 / (1 + 0.0183) ≈ 0.982

Step 3：0.982 > 阈值 0.5 → 预测 Fraud ✓

"Variable X (Distance) has a coefficient of β = +2.0, meaning that holding other variables constant, a 1-mile increase in transaction distance multiplies the odds of fraud by exp(2.0) ≈ 7.39 times. This is a strong positive predictor — the further the merchant is from the cardholder's billing address, the higher the fraud probability."

F-3 · 为什么 Accuracy 是骗子？（送分题，A+ 必背）

⚑

极端例子：100 笔交易里 1 笔欺诈、99 笔正常。模型全部预测"正常" → Accuracy = 99% ← 看似完美！但 Recall = 0% ← 一个欺诈都没抓到！

A+ 万能话术："In fraud detection, data is highly imbalanced (e.g., < 1% fraud). A naive model that predicts 'no fraud' for all transactions can achieve over 99% accuracy while detecting zero fraud. Therefore Accuracy is a misleading metric, and we should rely on Recall, Precision, F1-Score, and PR-AUC instead."

⑦ ROC vs PR-AUC — "阈值变了模型还行吗？"

G-1 · 阈值（Threshold）的概念

逻辑回归输出的是概率（如 0.7），但要做决定就得设个阈值（如 > 0.5 判欺诈）。

  • 阈值降低 → 更多东西被判欺诈 → TP↑ FP↑（抓得多但冤枉得多）
  • 阈值升高 → 更少东西被判欺诈 → TP↓ FP↓（保守但漏报多）

ROC / PR 曲线就是画出"在所有可能的阈值下"模型的表现。

G-2 · 两条曲线对比

ROC 曲线

  • X 轴：FPR = 1 − Specificity = FP/(FP+TN)
  • Y 轴：TPR = Recall = TP/(TP+FN)

AUC 值含义：

  • AUC = 1：完美模型
  • AUC = 0.5：和瞎猜一样（对角线）
  • AUC < 0.5：比瞎猜还差

PR 曲线

  • X 轴：Recall = TP/(TP+FN)
  • Y 轴：Precision = TP/(TP+FP)

PR-AUC 含义：

  • 越高越好，1 = 完美
  • 对类别不平衡更敏感
  • 不平衡数据中比 ROC-AUC 更真实

G-3 · 必考辨析：什么时候用哪个？

维度ROC-AUCPR-AUC
看什么TPR vs FPRPrecision vs Recall
适合平衡数据不平衡数据
不平衡数据下会骗你（TN 太大 → FPR 看起来超低 → AUC 虚高）更真实（不依赖 TN）
关注整体类别区分能力"模型说是欺诈时有多可信"
"In fraud detection where data is highly imbalanced (positive rate < 1%), PR-AUC is a more honest and informative metric than ROC-AUC. The dominance of True Negatives makes FPR artificially low, inflating ROC-AUC even when the model produces many false alarms. PR-AUC, which depends only on TP / FP / FN (not TN), more faithfully reflects how usable the alerts are in production."

A+ 标准答案：选 C（None of the studies）。理由（必须答方法论问题，不然只算半分）：

  1. 样本量太小：66 和 106 家公司，根本不够训练一个稳定的预测模型。这种规模只配做"探索性分析"，不能上线。
  2. 样本不平衡 / 人工平衡都有问题：
    • 罗马尼亚研究 44:22，没有处理不平衡。
    • 中国研究 53:53 的"完美平衡"，但现实欺诈率可能只有 0.1%。人工平衡的训练样本会严重高估真实环境表现。
  3. 没有独立测试集：两个研究都没清楚交代"训练 / 测试划分"，性能可能完全是在训练集上算出来的（即过拟合）。
  4. 变量选择 Look-ahead Bias（前视偏差）：用"是否被处罚"作为标签，再回头去看"哪些财务比率有问题"——等于偷看答案。
  5. 外部效度低：罗马尼亚 2015–2019、中国 2017–2020，仅限单一市场单一时间窗，不能推广到其他市场或其他时间。
⚑

重灾区警告：考试很爱考"研究批判"题。看到"Should we deploy this model?"，绝大多数情况下答案是 No，从 样本量 / 平衡性 / 过拟合 / 变量泄露 / 外部效度 这五条里挑两到三条来打。

B-5 · ❓ PPT 提问 5（Slide 12）— "Which performance metrics for LR model is more useful for fraud detection?"

?

原题："Which performance metrics for LR model is more useful for fraud detection?
A. R-square / AIC / BIC  ·  B. Precision  ·  C. Recall  ·  D. Accuracy  ·  E. None of the above"

追问："What is your choice? Why?"

A+ 标准答案：选 E（None of the above）。这是 PPT 第 13 页 Explanation vs. Prediction 的核心思想：

  • A (R²/AIC/BIC) 是 解释类 (Explanation/Inference) 指标，回答的是"我的模型对手头数据拟合得好不好"，不是预测能力。
  • B (Precision) 单独看会被骗：你只标记"100% 确定是欺诈"的那 1 笔，Precision = 100%，但 Recall 极低，业务上漏掉一大堆。
  • C (Recall) 单独看会被骗：你把所有交易都标成欺诈，Recall = 100%，但 Precision 极低，业务上每天打 100 万通骚扰电话。
  • D (Accuracy) 在不平衡数据下是骗子（见 L6 "Accuracy 陷阱"）。

正确答案应该是组合指标：

  • PR-AUC（Precision-Recall 曲线下面积）：不平衡数据下的金标准。
  • F1-Score：单数比较时方便。
  • PR Curve 本身：用来看 Threshold 选哪个最划算。

B-6 · Tiered Evaluation Strategy（PPT p.14–15，必背框架）

Tier 1 · Business-Critical（主看）

  • Recall：我们的欺诈捕获率够不够保护业务？
  • Precision：误报率够不够低，运营上承受得住吗？
  • PR Curve / PR-AUC：不平衡数据的最优可视化，比 ROC-AUC 更可靠。
  • F1-Score：单数对比时用。

Tier 2 · Goodness-of-Fit（次看）

  • R²：仅用于在同一数据集上对比不同模型。Tier 1 没改进的话，R² 改进无意义。
  • AIC / BIC：用于模型选择（如 8 变量 vs 17 变量），惩罚不必要的复杂度。

③ 决策树 (Decision Tree) — 一个新手侦探

C-1 · 三种节点 + 三个核心决定（必背）

决策树就是不断问 yes/no 问题来分类。例如："交易金额 > $100,000 吗？" → 是 → "客户以前有过欺诈记录吗？" → 是 → 判定欺诈。

节点类型说明
Root 根节点所有数据从这里开始（包含全部 attributes）
Decision Nodes 决策节点每一次提问
Leaf / Terminal Nodes 叶节点最终的判定结果（fraud / non-fraud）
三个决定内容 · 例子
Splitting 分裂决定选哪个属性、在什么值上切（如 "金额 > $100,000？"）
Stopping 停止决定什么时候不再加节点（深度到 5、叶节点 < 10 个样本就停）
Assignment 分配决定叶节点判什么类（叶子里 80% 是 fraud → 判 fraud）

C-2 · Impurity 不纯度 — 怎么决定"分哪个属性"？

白话：节点里如果全是 fraud 或全是 non-fraud，叫"很纯"；一半一半就"很不纯"。我们要选一个属性，让分裂后两个子节点尽可能纯。

任务用什么衡量
分类树 (Classification Tree)Gini Impurity 或 Entropy 熵
回归树 (Regression Tree)Variance / MSE 均方误差

C-3 · 三个公式（必背）

EntropyH = − pgood·log₂(pgood) − pbad·log₂(pbad)
GiniG = 1 − (pgood² + pbad²)   ·   一般式: G = 1 − Σpc²
Info GainIG = Hparent − Σ (nchild/nparent) · Hchild
★

决策规则：选 IG 最大 的属性来分裂；或等价地，选 加权 Gini 最小 的属性。

C-4 · ⚡ 必考计算题：信息增益 IG 实战（PPT p.21–22）

·worked example · 父节点 Fraud=40, Non-Fraud=80（共 120）

Step 1 · 算父节点 Entropy：H = −(40/120)·log₂(40/120) − (80/120)·log₂(80/120) = 0.918

方案 A · 用 AGE > 45 分：

  • 子节点 1（NO，70 个）：Fraud=28, Non-Fraud=42 → H = 0.971
  • 子节点 2（YES，50 个）：Fraud=12, Non-Fraud=38 → H = 0.795
  • IG(AGE) = 0.918 − (70/120)·0.971 − (50/120)·0.795 = 0.02

方案 B · 用 INCOME > 10,000 分：

  • 子节点 1（NO，40 个）：Fraud=10, Non-Fraud=30 → H = 0.813
  • 子节点 2（YES，80 个）：Fraud=30, Non-Fraud=50 → H = 0.954
  • IG(INCOME) = 0.918 − (40/120)·0.813 − (80/120)·0.954 = 0.01

结论：IG(AGE) > IG(INCOME) → 选 AGE 作为分裂属性 ✓

C-5 · 同样的题用 Gini 再算一遍（PPT p.52）

·worked example · Gini Impurity 验证

方案 A · AGE > 45：

  • 子节点 1：G = 1 − (28/70)² − (42/70)² = 0.480
  • 子节点 2：G = 1 − (12/50)² − (38/50)² = 0.365
  • 加权 Gini = 70/120 × 0.480 + 50/120 × 0.365 = 0.432

方案 B · INCOME > 10,000（题目重新分配为 35/85）：

  • 加权 Gini = 35/120 × 0.408 + 85/120 × 0.457 = 0.443

结论：Gini(AGE) < Gini(INCOME) → 同样选 AGE ✓（Gini 越小越好，与 IG 越大越好等价）

④ 决策树的问题 — 3 个 PPT 提问

D-1 · ❓ PPT 提问 6（Slide 25）— "What are the possible problems with Decision Tree model?"

?

原题："What are the possible problems with Decision Tree model?"

A+ 标准答案 — 三大缺点必背（PPT p.26 + p.29）：

  1. 过拟合 (Overfitting) — 最大问题：树长得太深、太复杂，把训练数据的噪音也学进去了。后果：训练集准确率 99%，测试集 65%。
  2. 不稳定 (Unstable)：训练数据变一点点（哪怕换一行），整棵树的分裂结构都可能完全不同。
  3. 连续变量被强行离散化：一刀切成"金额 > $100k 或 ≤ $100k"，丢失了金额的连续信息。

解决方案（必背两条路）：

方案子方案说明
A) Pruning 剪枝Pre-pruning 预剪枝预定义 stop 条件（最大深度 / 最小样本数 / 最大叶节点数），到了就停，不让树长完整
Post-pruning 后剪枝先 fully-grown 完整长成 → 用 Validation Set 反向砍掉对验证集没用的枝
B) Ensemble MethodsBagging / Boosting / RF下一节讲，是真正的大招

D-2 · Post-pruning 流程（PPT p.27，必背）

  1. 把数据切成 Training Set（通常 70%）和 Validation Set（Training 的 30%）。
  2. 让树 fully-grown 完整长成。
  3. 反向回剪，直到 Validation Set 的 misclassification error 降到最小 时停手。

⑤ Ensemble Learning — "招一队侦探"

E-1 · 是什么 + 为什么

白话：与其相信一个新手侦探，不如让 一群侦探（base models / weak learners） 各自破案，最后投票决定。

  • Performance（性能）：群体智慧比任何单个个体更准。
  • Robustness（鲁棒性）：即使某几个侦探判断错，多数投票能纠正，预测分散度更小。

E-2 · 同质 vs 异质（必背）

  • Homogeneous Ensemble 同质：所有侦探都是同一类型（如全是决策树）。
  • Heterogeneous Ensemble 异质：侦探类型不同（决策树 + LR + SVM），需要训练一个"主管"模型来综合大家的意见。不同基模型的选择必须和聚合方式一致。

E-3 · 三种组合方式 ⭐ 必考对比表（PPT p.33）

方法同/异质训练方式组合方式目标例子
Bagging同质并行（独立训练）简单平均 / 多数投票降方差 (variance)Random Forest
Boosting同质顺序（后一个依赖前一个）加权组合（确定性策略）降偏差 (bias)AdaBoost / GradientBoost
Stacking异质并行训练一个 meta-model 综合整体性能—
★

记忆口诀：
· Bagging = Bootstrap + Aggregating，并行投票（降方差）。
· Boosting = 后浪推前浪，串行修正（降偏差）。
· Stacking = 堆叠不同模型 + 上面再加一个总指挥。

E-4 · Bootstrap Sampling（必背概念）

白话：从原数据集里 有放回地 重复抽样，得到很多个新数据集。

  • 依据：大数定律——抽样次数够多，样本就能代表总体。
  • 为什么要有放回？ 如果样本能代表总体，则总体里应该有多个相似观测；有放回正好模拟这一点。

Bootstrap 流程（PPT p.35，5 步）：

  1. Step 1 决定 bootstrap 样本数量（如 100 份）。
  2. Step 2 决定每份样本大小。
  3. Step 3 对每份：有放回抽样 → 算统计量。
  4. Step 4 把所有样本的统计量取平均。
  5. Step 5 （在 Bagging 里）每份样本训练一个 base model，最后投票/平均。

⑥ Random Forest — "侦探小队的终极形态"

F-1 · 三步流程（必考）

  1. Step 1 · Bootstrap 抽样：从原始数据集有放回抽样得到一个新训练集。
    举例：原 5 个观测、4 个变量 → bootstrap 后可能 ID=4 出现两次，ID=5 一次也没出现。
  2. Step 2 · Random Feature Subset：训练一棵决策树，但 每个节点只随机抽 m 个特征 里选最好的来分（不用全部特征！）。
    m 的常见选择：m = 1, 2, 或 log₂(N)+1（N 是变量总数）。
  3. Step 3 · 重复 T 次，得到 T 棵不同的树 → 组成森林。
    · 分类：T 棵树多数投票。
    · 回归：T 棵树取均值。
⚑

必背：Random Forest 的"随机"来自两层：
① 每棵树用的训练样本不同（Bootstrap）
② 每个节点考虑的特征子集不同（Random Subset of Features）
两个随机加起来 → 树之间差异够大 → 投票才有意义。这就是 RF 比单棵 DT 好的根本原因。

F-2 · Out-of-Bag (OOB) Sample（必考概念）

白话：因为是有放回抽样，每次大约有 1/3 的原始数据没被抽到——这些被冷落的样本叫 OOB。

  • 每个 OOB 样本，会被丢进 所有"没看过它"的树 里去预测。
  • 拿这些预测和真实标签比对 → OOB Score = OOB 样本中被正确预测的比例。

F-3 · ❓ PPT 提问 8（Slide 44）— "Can we use OOB score to evaluate RF? When is it useful?"

?

原题："Can we use OOB score to evaluate the performance of the Random Forest model?
When is OOB score be useful?"

A+ 标准答案：Yes。OOB Score 可以用来评估 RF，因为 OOB 样本对每棵树来说都是"没见过"的数据，类似一个内置的验证集。

OOB Score 特别有用的 3 个场景：

  1. 样本量很小：当数据少到舍不得划出独立的 Validation/Test Set 时，OOB Score 让你不用切分数据就能估计泛化性能。
  2. 快速调参：调 ntree、mtry 时，OOB 比 Cross-Validation 快得多。
  3. 节省计算：不需要额外做 K-fold CV，因为 OOB 本质上已经做了类似的事。
!

但仍然建议保留独立 Test Set 做最终评估——OOB 只在训练循环中用，不能取代真正的测试集。

F-4 · Random Forest 优缺点

✅ 优点

  • 预测性能极好，非常适合欺诈检测
  • 适合少观测、多变量的数据集
  • 自带 OOB 估计，省去 CV 步骤

❌ 缺点

  • 黑盒模型 (black-box)，可解释性差
  • 解决方法：用 Variable Importance 来打开黑盒（下一节）

⑦ Variable Importance — "打开黑盒的翻译官"

G-1 · 两种解释方式（必背对比）

Overall Interpretation 整体解释Local Interpretation 局部解释
回答什么哪些变量整体上最有预测力？这一笔交易，为什么被判定为欺诈？
用法特征筛选、模型对比给客户/审计员解释单笔判断
不平衡数据下—能找出"最频繁的预测原因"，特别有用

G-2 · Variable Importance 的 4 大用途（必背）

  1. 高重要性变量 → 进一步分析、保留。
  2. 低重要性变量 → 可以丢弃，简化模型。
  3. 某变量重要性高得离谱 → 可能是 数据泄露 / Bug（比如不小心把 label 当 feature）。
  4. 比较加新变量前后的 Importance → 判断新变量有没有用，或对比不同模型。

G-3 · 两种计算方法 ⭐ 重灾区，必考对比

方法 1 · MDI

Mean Decrease in Impurity（均值不纯度下降）

原理：每次某变量被用来分裂节点，都让 Gini 下降一点；把所有树、所有节点的下降量加总平均。
· 计算时按样本数加权。
· 在分裂条件 (split-criterion) 上的改进就归功于该分裂变量，并在所有树上累加。

✅ 计算快、便宜。
❌ 偏向连续变量和高基数 (high cardinality) 变量——值越多越容易"看起来很重要"。

方法 2 · MDA (Permutation)

Mean Decrease in Accuracy（均值准确率下降，亦称 Permutation Importance）

4 步流程：

  1. 用 OOB 样本算每棵树的准确率（baseline）
  2. 随机打乱 第 j 个变量的值（其他变量不动）
  3. 用打乱后的 OOB 样本再算一次准确率
  4. 准确率下降越多 → 该变量越重要

✅ 更可靠、不偏向高基数。
❌ 计算慢。

MDA 公式VIj = (1/T) · Σt=1..T ( Acct − Acctpermuted(j) )
i

R 包：caret::varImp() 通用；randomForest::varImpPlot() 专门画 RF 的。Variable Importance 不仅适用于 RF，也适用于其他 ensemble 模型。

H-4 · ❓ PPT 提问 9（Slide 69）— "Should importance of predictors be assessed?" + "P/R/F-score on TRAINING data?"

?

原题 (Q9a)："Should importance of predictors be assessed?"

原题 (Q9b)："What are the Precision, Recall & F-score of this 'TRAINING Data'?"

Q9a 答案：应该评估。理由：

  1. RF 是黑盒，Variable Importance 是唯一窗口让我们看到模型在用哪些变量做决策。
  2. 业务可解释性：审计师 / 监管要看模型逻辑。
  3. 帮助特征筛选：去掉低重要性变量，模型更轻、更快、更稳。
  4. 检查数据泄露：某变量异常重要可能是 bug。

Q9b 答案：不要被训练数据上的指标骗了。RF 在训练数据上几乎都是接近 100% 的（因为 bootstrap + ensemble 让它能"记住"训练样本）。真正有意义的 P/R/F1 必须在测试数据上算。这道题真正的考点是："训练集表现完美 ≠ 模型好"，要警惕过拟合。

H-5 · ❓ PPT 提问 10（Slide 70）— "Training >> Testing 怎么解释？"

?

原题："What does it mean when the performance results of training data is MUCH better than testing data?"

A+ 标准答案：典型的过拟合 (Overfitting)。

  • 现象：模型把训练数据的噪音 + 偶然规律都背下来了，但学不到真正可推广的规律。
  • 类比：学生把习题册答案背得滚瓜烂熟，一上考场题目稍微变一下就不会了。
  • 常见原因：① 模型太复杂（树太深 / mtry 太大）；② 训练样本太少；③ 特征里有数据泄露。
  • 解决：① 决策树 → 剪枝；② 随机森林 → 调小 mtry、限制 max_depth、加更多 bootstrap 样本；③ 通用方法 → 加正则化、加更多数据、做 K-fold CV。

H-6 · ❓ PPT 提问 11（Slide 71）— "How to interpret the PRC value?"

?

原题："How to interpret the PRC value?"

A+ 标准答案：

  • PRC = Precision-Recall Curve（精确率-召回率曲线），PRC AUC = 它下方的面积。
  • 横轴 Recall：抓到了多少比例的真欺诈。
  • 纵轴 Precision：被你抓的人里有多少是真欺诈。
  • 取值：0 ~ 1（越高越好）。
  • 业务解读：PRC AUC 接近 1 → 模型能在高 Recall 的同时维持高 Precision，对欺诈检测来说是理想状态。
  • 不平衡数据下，PRC AUC 比 ROC AUC 更可信——因为 ROC AUC 会被海量负样本"骗"得很高。
⚑

必背对比：
· 平衡数据 / 关心两类都准 → 看 ROC AUC。
· 不平衡数据 / 主要关心少数类（欺诈）→ 必看 PRC AUC。

⑨ K-fold Cross-Validation — "小数据怎么办？"

I-1 · ❓ PPT 提问 14（Slide 81）— "How to handle the dataset when the sample size is small?"

原题："How to handle the dataset when the sample size is small?"

A+ 标准答案：用 K-fold Cross-Validation（K 折交叉验证）。当样本量 < 1000 时考虑 CV，< 100 时考虑 LOOCV。

I-2 · K 折交叉验证是什么？

白话：把数据切成 K 份，轮流用其中 1 份当测试集、其余 K−1 份当训练集，跑 K 次，把 K 次的误差取平均。

轮次Fold 1Fold 2Fold 3Fold 4Fold 5
1测训训训训
2训测训训训
3训训测训训
4训训训测训
5训训训训测

最终性能 = 5 次测试误差的平均值。

I-3 · K 怎么选？（PPT p.83，必背）

K 值名称特点
K = 5 或 10标准做法低偏差 + 中等方差，业界默认
K = nLeave-One-Out CV (LOOCV)每次只留 1 个观测做测试，特别适合极小数据集，每个样本都有机会做测试
选 K 的关键原则：每个 train/test fold 都要在统计上代表 (representative)整个总体。

I-4 · ❓ PPT 提问 15（Slide 84）— "With more than one trained model, choose which model?"

?

原题："Q : With more than one trained model, choose which model?"

A+ 标准答案 — PPT 给了 3 种策略：

  1. Voting（投票法，类似 Ensemble）：让 K 个模型一起投票，最终预测取多数。
  2. LOOCV + 随机选一个：因为 LOOCV 下每个模型只差一个观测，性能几乎一致，随便选一个都行。
  3. All-Data Train + CV Performance as Estimate：用全部数据重新训一个最终模型，把 CV 的平均性能作为这个最终模型的独立性能估计。
★

第 3 种是最常用的工业实践——CV 用来"估性能"，最终模型用全部数据训练。

④ Back Testing 三件套（必背）

D-1 · Data Stability — SSI 红绿灯

白话：上线之后输入数据的"长相"是不是变了？比如训练时全是 25-35 岁客户，上线后突然涌入大批 60+ 老人，模型就废了。

两步法：

  1. Step 1：当前用模型的人群 vs 训练时人群 → 整体相似？
  2. Step 2：若不像，逐变量看哪个特征飘了。

SSI（System Stability Index）红绿灯：

SSI 值信号含义 + 行动
SSI < 0.10🟢 绿灯没显著偏移，继续用
0.10 ≤ SSI < 0.25🟡 黄灯中度偏移，持续监控 + 准备重训
SSI ≥ 0.25🔴 红灯显著偏移，立刻重训

⚠️ 注意：SSI 也叫 PSI（Population Stability Index），两个缩写在 L9 是同一个东西，考试两个都可能出现。

D-2 · Model Stability — 性能漂移红绿灯

白话：模型的"准确率 / Recall"在不同时间段还稳吗？还是只在某段时间灵？

判断标准（必背）：跟踪关键指标（如 Recall / F1）的绝对变化：

变化幅度信号含义
< 5%🟢 绿正常波动
5% ~ 10%🟡 黄留意，可能开始退化
> 10%🔴 红明显退化，触发重训

D-3 · Model Calibration — 概率校准 🔥 最容易出大题

白话：模型说"这笔有 30% 概率是欺诈"，实际真的有 30% 是欺诈吗？

6 步流程（必背）：

  1. Partition Data Temporally：按时间切（如滚动月窗口）划分历史训练/测试集。
  2. Score Transactions：在"未来"样本上预测欺诈概率。
  3. Bin Predictions：按预测概率分组 0.0–0.1 / 0.1–0.2 / … / 0.9–1.0。
  4. Calculate Observed Rates：每组里算实际欺诈比例。
  5. Plot Calibration Curve：画"预测概率 vs 实际欺诈率"图，完美校准 = 45° 对角线。
  6. Compute Metric：用 Brier Score 或 Expected Calibration Error (ECE) 量化。Brier 越接近 0 越好。
·worked example · 信用卡欺诈 0.6–0.7 概率桶

设置：1 月份信用卡交易，模型在 0.6–0.7 桶预测了 50 笔，其中 31 笔实际是欺诈。

· 平均预测概率：0.65
· 实际欺诈率：31 / 50 = 0.62
· 差距：0.65 − 0.62 = 0.03 → 校准良好，接近对角线 ✓

三种状态对照：

状态曲线位置业务后果
Well-calibrated
校准良好		 贴近 45° 对角线 调查阈值可信，资源分配最优
Over-confident
过度自信		 曲线在对角线下方（预测 > 实际） 过度警报 → 浪费调查人力
Under-confident
过度保守		 曲线在对角线上方（预测 < 实际） 低估风险 → 真欺诈漏掉
⚑

Brier Score 必背公式：Brier = (1/N) Σ (pi − yi)²
其中 pi 是预测概率，yi 是 0/1 实际标签。越接近 0 越好。

⑤ Adversarial Testing 三件套（必背）

E-1 · Adversarial Examples — 对抗样本

白话：坏人把一笔欺诈交易微调几个数字（金额改几块、时间挪几分钟、加点 GPS 噪音），模型就识别不出来了。人眼看几乎没变，模型却判若两人。

核心算法（必记名字）：HopSkipJump Attack / Boundary Attack（边界攻击）。

  1. Initialization：随便找一个能骗过模型的对抗样本起步。
  2. Binary Search：在合法 / 欺诈两个区域之间，二分逼近决策边界。
  3. Gradient Estimation：不需要看模型内部，靠多次查询估梯度方向。
  4. Boundary Walk：沿着边界往目标输入慢慢挪。
  5. Step Size Optimization：用几何级数找最优扰动幅度。
·worked example · 信用卡欺诈样本被改装

原本：金额 $850 · 凌晨 2:30 · 加油站 · 异常城市 → 模型欺诈得分 0.95（红牌警告）

HopSkipJump 改装：
· Amount：$850 → $847（-$3）
· Time：2:30 AM → 2:45 AM（+15 分钟）
· GPS：加一点点坐标噪音

结果：得分 0.20（远低于 0.5 阈值）→ 模型放行，攻击成功 ✗

E-2 · Model Poisoning — 模型投毒

白话：坏人在模型训练阶段就下手，把"毒药数据"混进训练集，从源头让模型学错。

类型目的方法例子
Indiscriminate
无差别投毒		 让模型整体变废 注入随机噪音 / 无意义数据 加 1000 条乱标签数据，模型失去判别力
Targeted
定向投毒		 让模型在特定类型上失灵 战略性注入造特定失效模式 注入"合法外观+特定商户码"的样本标记为欺诈，让模型对该商户码全部误判
Backdoor
后门投毒		 留个只有攻击者知道的密码 埋藏只在特定触发器下激活的恶意行为 训练模型把所有金额结尾 ".37" 的交易都标为合法，不管其他特征多可疑
$

必背案例数字：银行模型被定向投毒后，Recall 从 95% → 73%，攻击检测率几乎为零。

E-3 · Transferability Property — 对抗样本的迁移性 🔥 最颠覆的考点

白话：坏人根本不用碰你的银行系统！他自己训一个类似的模型，针对自己的模型生成对抗样本，然后这些样本也能骗过你的模型！

为什么会迁移：

  • Shared Decision Boundaries：相似数据训练 → 学到相似决策面
  • Feature Space Vulnerabilities：对抗扰动利用特征表示的根本弱点
  • Common Training Patterns：相似训练流程 → 相似行为
!

毁灭性数字（必背 94%）：在信用卡欺诈检测里，Logistic Regression 和 Random Forest 之间的对抗样本迁移成功率 = 94%。说明 —— 集成多模型不一定安全！

The Ensemble Vulnerability Paradox（集成脆弱性悖论）：

  • 传统想法："多个不同模型投票 = 通过冗余获得安全。"
  • 现实：迁移性可以同时攻陷整个集成。
  • 研究证据：基于梯度的攻击竟然也能骗过非梯度的树模型；脆弱性跨算法家族传播（NN / 树 / 线性都中招）。
·worked example · 多模型集成被一网打尽（课件 Slide 66）

银行部署：XGBoost + Neural Network + Logistic Regression 三模型，≥2 个判定为欺诈才报警。

攻击者操作：
① 自己用类似数据训一个 替身 XGBoost（Surrogate）
② 针对替身生成对抗扰动
③ 测试 → 部署到真实交易

结果：原本 3 个模型一致判为欺诈 → 修改后 0 个模型报警，完全突破。94% 的对抗样本骗过所有集成成员。

⑦ Q3 衍生题 — A+ 必备话术

① "Recall 跌了，3 个原因 + 诊断"（2025 Dec Q3.4 已考）

"Recall degradation can be explained by three concurrent root causes:
① Concept Drift: fraudsters change tactics — historical fraud signature no longer matches new fraud.
Diagnose: compute SSI on input features over time; SSI ≥ 0.25 in any feature confirms drift.
② Data Quality / Pipeline issue: a feature is silently null, a new merchant category appears.
Diagnose: monitor null rate, distinct-value count, range vs training baseline (data-health dashboard).
③ Adversarial Evasion: fraudsters craft transactions scoring just below the threshold.
Diagnose: inspect score distribution of missed fraud — clustering below cut-off = adversarial; run HopSkipJump / Boundary Attack simulations.
Mitigation: scheduled retraining (monthly or when SSI ≥ 0.25), online learning, A/B testing new models, human-in-the-loop for high-amount alerts."

② "Calibration 突然崩了 / 看到 Over-confident 曲线"

"The calibration curve lies below the 45-degree diagonal, indicating the model is over-confident (predicted probabilities exceed observed fraud rates). Consequence: excessive false alarms → wasted investigator effort and customer friction. Numeric assessment: Brier Score (MSE between predicted probabilities and observed outcomes); closer to 0 = better. Remediation: recalibrate with Platt scaling or isotonic regression on a recent out-of-sample window, then re-validate against the diagonal."

③ "为什么集成模型不一定能防对抗攻击 / 是否要堆叠更多模型"

"Stacking more diverse models does not guarantee robustness due to the Transferability Property: adversarial examples crafted against one model often succeed against entirely different architectures because they share comparable decision boundaries and exploit feature-space vulnerabilities. Empirical evidence: 94% transferability rate between Logistic Regression and Random Forest in credit-card fraud. This is the Ensemble Vulnerability Paradox — redundancy creates a false sense of security. Genuine defenses require adversarial training, input preprocessing (denoising), certified defenses, and human-in-the-loop review for borderline cases, not just more models."

④ "AML 系统为什么 90% 假阳率"（短题 / MCQ）

"Traditional rule-based AML systems generate ~90% false positive rates (some up to 95%), costing institutions an average of $12.9M annually in unnecessary investigations. Root causes:
① rigid rule-based logic that cannot adapt to evolving laundering patterns;
② lack of contextual understanding of customer behaviour and business operations;
③ outdated rules generating irrelevant alerts.
Modern remediation combines rule-based detection for known risks with supervised ML and unsupervised anomaly detection for novel patterns, plus privacy-preserving cross-bank collaboration via Federated Learning, Secure Multi-Party Computation, and Differential Privacy."

① 为什么要学 SNA？— Guilt-by-Association（连坐推定）

课件原话："物以類聚 – Birds of same feathers flock together." —— 如果 Tom 的好朋友全是罪犯，Tom 大概率也不干净。整节课所有指标，本质都是在量化这一句话。

  • 传统欺诈检测的盲区：只看单个账号 / 单笔交易，看不到"一伙人"的协同作案。
  • SNA 解决的痛点：保险骗保（同一证人 / 同一索赔人重复出现）、公司破产链（同一股东 / 同一员工反复操盘）、虚假评论（同一 IP / 同一设备群刷）。
  • Small-world (六度分隔)：Milgram 1967 实验显示真实社会平均路径长度 ≈ 6 跳；Kwak 2010 显示社交网络上仅 ≈ 4 跳 —— 欺诈传播比你想象快得多。

② 网络的"砖头" — Graph Theory 基础术语

边的 5 种形态（必背）

  • Directed（有向）：A 转账给 B，有方向
  • Undirected（无向）：A 和 B 是朋友
  • Self-edge（自环）：自己给自己转账 —— 洗钱常见手法
  • Multi-edge（多重边）：同一持卡人在同一商家刷了 5 次卡
  • Hyper-edge（超边）：3 人同时参加一场会议

边的权重 3 种类型

  • Binary（0/1 或 -1/0/1）：有无关系 / 敌友路人
  • Numeric（数值）：数字越大关系越铁（共同活动数 / 消息数）
  • Normalized（归一化）：一个节点所有出边权重和 = 1
Jaccard 权重 wJaccard(A,B) = |共同事件| / (|A的事件| + |B的事件| − |共同事件|)

小例子带你算一遍：A 参加 10 个活动，B 参加 5 个，共同 3 个 → Jaccard = 3 / (10+5−3) = 0.25（重合度 25%）。这个权重的好处：避免被"高交际花"干扰 —— 一个谁都认识的人，跟你共 3 个活动其实没那么"铁"。

③ Egonet（自我网络）— 课件最爱考的一个词

  • 定义：一个节点 + 它的 一跳邻居 (1-hop neighbours) + 这些邻居之间所有的边。
  • 类比：你的"朋友圈截图" —— 只看你和直接好友。
  • 课件原话（必背送分句）："One-hop is enough for fraud analysis." —— 抓骗子用一跳就够了，不需要扩到二跳。

⑤ Degree 进阶 — Fraud Degree vs Total Degree（A+ 必懂套路）

★

课件 p.37 经典对比题：

  • Node A：Degree = 5（5 个全是骗子）→ Fraud degree = 5（占比 100%）

正确答案：A 更可疑。"Total degree alone is a misleading signal — fraud degree (and its proportion) is what matters for guilt-by-association."

⑥ Triangles 三种形态（必背术语）

  • Fraudulent triangle：3 个节点全是骗子 → 铁三角骗团
  • Semi-fraud triangle：3 个中有 1 个是骗子 → 嫌疑人圈
  • Legitimate triangle：3 个都清白 → 正常社交圈

课件 A+ 信号句（直接抄进答卷）："High fraudulent degree + High density ⇒ High probabilities to commit fraud in future."

⑦ Relational Neighbour Classifier — 同质性 (Homophily) 投票

公式P(c | n) = (1/Z) × Σ w(n, nj) · 1[nj ∈ c]

小例子（课件 p.31）：Node G 有 4 个邻居，3 个 Fraud + 1 个 Non-Fraud，权重全为 1。
P(F|G) = ¼ × (1+1+1) = 0.75；P(NF|G) = ¼ × 1 = 0.25 → 结论：G 大概率是骗子。

Probabilistic 版本：把上面的 "1（是骗子）" 替换为 "0.7（70% 概率是骗子）" 即可 —— 邻居自己也是别的模型预测出来的概率。

⑨ PageRank — 把"网页排名"改造成"欺诈传播"

PageRankPR(A) = (1−α)·eA + α · Σi→A [ PR(i) / Dout,i ]
  • α ≈ 0.85：用户顺着链接走的概率
  • 1−α ≈ 0.15：用户烦了"random surfer"随机跳页的概率
  • eA：重启向量（关键改造点）

欺诈版本（PageRank_fraud）的核心 trick：

  • 已知骗子节点：重启向量 ei = 非 0（"注入欺诈源"）
  • 清白节点：ei = 0
  • 跑完算法后：每个节点的 PR 值 = "它被骗子影响多深"

⑩ SNA 的 5 大局限（A+ 收尾必写）

  1. Guilt-by-association 不是法庭证据：只能 trigger investigation，不能定罪。
  2. 数据质量决定一切：节点 ID 缺失、关系错连 → 整张图崩。
  3. 隐私 / 法律风险：抓关系图可能涉及 GDPR / PDPO，要合规。
  4. 计算量爆炸：百万节点的 PageRank / Betweenness 算起来非常贵。
  5. 静态快照 vs 动态网络：关系会变化，单一时点的图可能误导（要加时间维度）。

📖

原文场景（课件保留）：

  • 在这个案例中，我们看节点 "Suzie Smith"。
  • Suzie Smith 与一家已倒闭的问题公司有直接关系（direct relationship） —— 这是一跳关系（1-hop）。
  • 同时她还与另一家已倒闭的问题公司有二度关系（second-degree relationship） —— 即邻居的邻居，是两跳关系（2-hop）。

课件配套思考题："What is the PROBLEM of this network?"

  • 表层问题：Suzie 的关系网里出现了 2 家已倒闭的问题公司（1 跳 + 2 跳）—— 单点信号已经触发红旗。
  • 深层问题：这是典型的"借壳重启"骗局结构 —— 同一批人换不同壳公司反复经营、避税、逃债。SNA 在注册阶段就能截住。
  • SNA 在这里用了哪些指标？
    • Degree + Fraud degree：邻居中的"问题公司"数量
    • Relational Neighbour Classifier：根据邻居标签直接推 Suzie 的欺诈概率
    • 2-hop egonet：拓展到二度关系才看见全图（注意课件原话"One-hop is enough"是针对核心欺诈分析；调查阶段可放宽到 2 跳找隐藏关系）

为什么这是 SNA 的"成本优势卖点"？

💰

事后稽查 vs 注册阶段拦截 的成本对比：

  • 事后稽查：要派税务专员实地走访、调银行流水、约谈相关方 —— 人力 + 时间 + 法律风险全部叠加。
  • 注册阶段 SNA 拦截：只需把申请人信息匹配现有黑名单图谱，一秒返回风险评分 —— 边际成本接近 0。
  • 结论："This could trigger the state to deny the new registration or conduct additional investigations in a cost-effective manner."（课件原句）

A+ 答题模板

"The FICO tax-fraud example illustrates the cost-efficiency advantage of SNA. Suzie Smith's egonet reveals a direct (1-hop) link to one defunct problem company and a 2-hop link to another, indicating a likely 'shell-recycling' pattern where the same actors repeatedly close and re-register entities to evade tax obligations. By scoring incoming registrations against a known-fraud network, the tax authority can deny or flag suspicious applications at the registration gate, which is materially cheaper than post-hoc audits — substituting near-zero marginal compute cost for expensive on-site investigation. 课件 p.70–77 原案例。原始论文：Fronzetti Colladon, A. & Remondi, E. (2017). Using social network analysis to prevent money laundering. Expert Systems with Applications, 67, 49–58. DOI: 10.1016/j.eswa.2018.09.029

① 背景（课件原文保留）

📖
  • 数据来源：意大利一家中大型保理公司（factoring company）的中央数据库。
  • 覆盖期间：2013 年 11 月 — 2015 年 6 月的所有金融操作记录。
  • 法律规定："According to the Italian law, each financial institution has to record all the transactions amounting to at least 15,000 euros and smaller transactions in case their aggregated value is bigger or equal to 15,000 euros (to face the problem of smurfing)."
  • 翻译：≥ €15,000 单笔必报；< €15,000 但聚合后 ≥ €15,000 也必报 —— 专门防"smurfing"（化整为零、拆小金额逃避监管）。

② Fraud Data Analytics 的 Scope 定义

课件 p.72 明确：本研究关注的"洗钱嫌疑交易"是 —— debtor → factor 的资金转移操作（这是把"脏钱"放进"干净循环"的核心环节）。

1) 构建交易网络

  • 节点：seller & debtor
  • 同时构建 3 种类型的网络：

    • Economic sector network（行业网络）

④ 分析方法

课件 p.75 原文："Several logit models are tested to assess the predictive power of the SNA metrics" —— 跑了多个 Logistic Regression（逻辑回归）模型，逐步加入 SNA 指标作为自变量，比较模型预测能力。

课件给出的两个对照模型：

  • Model 1：只用 Missing ID（baseline）→ R² = 0.126
  • Model 4：加入所有 SNA 网络特征（Degree / Density / Centrality 等）→ R² = 0.327

⑤ 课件配套思考题 a："How to interpret the predictive value of SNA using this table?"

  • 正向解读：R² 从 0.126 → 0.327 翻了一倍多，说明 SNA 网络特征对洗钱预测有显著增量贡献。
  • 关键变量：能让 R² 大幅提升的，通常是 Closeness / Betweenness（找中介）和 Triangle 数量（找团伙）—— 因为洗钱本质就是"团伙 + 中介"的协同行为。
  • 系数符号：需要看 logit 模型中各 SNA 变量的系数（β）和显著性（p-value）：β > 0 且 p < 0.05 → 该指标越高，洗钱概率越大。

⑥ 课件配套思考题 b："How to interpret the goodness of fit measures? (R², AIC, BIC)"

指标含义判断方向
R²模型解释了多少比例的因变量变化（0–1）越大越好；但加变量必涨，需配合 Adjusted R²
Adjusted R²R² 减去"变量数惩罚"越大越好；能反映过拟合
AIC
Akaike Information Criterion		−2·log-likelihood + 2k（k = 变量数）越小越好；偏向预测准确
BIC
Bayesian Information Criterion		−2·log-likelihood + k·log(n)（n = 样本数）越小越好；对变量数惩罚更重，偏向简洁模型

💡 记忆口诀："R² 越大越好，AIC / BIC 越小越好。" BIC 比 AIC 更严厉惩罚多变量，所以 BIC 更倾向 "简洁模型"。

⑦ 课件 p.77 经典 A+ 陷阱题（重点中的重点）

⚠️

题目原文（课件保留）：

"Model 1 (Missing ID only): R² = 0.126
Model 4 (All network features): R² = 0.327
Question: The R² more than doubled (0.126 → 0.327). Does this mean network analysis is 'better' than simple rule-based checks? What are you not seeing in this table?"

⑧ A+ 答案模板（课件就是要你说出陷阱）

"R² doubled but is still only 0.327, meaning 67% of the variation in money-laundering behaviour remains unexplained. Crucially, the table does not show:
(1) Adjusted R² which penalises added variables — Model 4 has many more predictors than Model 1, and R² will mechanically rise even with noise variables;
(2) Out-of-sample / cross-validated performance — in-sample R² says nothing about generalisation, which is what matters for deployment;
(3) Precision / Recall / PR-AUC which matter far more than R² for highly imbalanced fraud data — money-laundering events are rare, so a model can have decent R² yet still miss almost all true cases;
(4) Statistical significance (p-values) of individual SNA coefficients — without p-values we cannot tell which network features genuinely drive the prediction versus which are along for the ride.
Higher R² is encouraging but not sufficient evidence that SNA is 'better' than rule-based checks; it is merely a red flag for further validation."
🔗

知识串联点：这道题同时考察 L6 (Adjusted R²) + L7 (Precision/Recall/PR-AUC) + L10 (SNA)。也就是说，L10 这道陷阱题就是把你之前学的东西串起来考。如果你前两个问题搞清楚了 Adjusted R² 和 PR-AUC 是什么，这道题等于送分。

⑨ 实战启示（写论述题可加分）

  • SNA 不是孤立技术：必须和 logit / Random Forest 等传统监督模型结合 —— SNA 提供"网络特征"作为额外输入，而非完全替代。这就是 GOTCHA 案例反复强调的 "local + network features combined outperforms either alone"。
  • 3 种网络互补：Transactions / Sector / Geography 三张图各抓不同维度的洗钱模式 —— 单一网络容易漏看（如只有交易网会漏掉"同行业互相走单"的协同模式）。
  • Smurfing 反洗钱机制：意大利法规明确要"聚合 < €15k 的小额"也要报告，这正是立法层面对 SNA 思维的认可 —— 单笔小额看不出问题，关系网聚合才能暴露。

Q1.1 Gini 不纯度计算（10 分）

题目：给 10 个交易（Insurance Claimed Amount, Annual Income, Fraud=YES/NO）。
(a) 用 "Insurance Claimed Amount >= 100k" 作为分裂条件，算 Gini。
(b) 用 "Annual Income <= 50k" 作为分裂条件，算 Gini。
(c) 哪个属性当分裂节点更好？

原始数据统计：10 笔交易中，3 笔欺诈（YES）（ID 1, 7, 9）+ 7 笔正常（NO）。

父节点 Gini = 1 − (3/10)² − (7/10)² = 1 − 0.09 − 0.49 = 0.42

(a) 按 Insurance Claimed Amount >= 100k 分

  • ≥ 100k 一组（金额 ≥ 100k）：ID 1, 2, 3, 5, 6, 7, 8 → 7 人，其中 Fraud=YES：ID 1, 7 → 2 YES, 5 NO
    Gini = 1 − (2/7)² − (5/7)² = 1 − 0.0816 − 0.5102 = 0.408
  • < 100k 一组（金额 < 100k）：ID 4, 9, 10 → 3 人，其中 Fraud=YES：ID 9 → 1 YES, 2 NO
    Gini = 1 − (1/3)² − (2/3)² = 1 − 0.111 − 0.444 = 0.444
  • 加权 Gini = (7/10)(0.408) + (3/10)(0.444) = 0.286 + 0.133 = 0.419
  • ΔGini(a) = 0.42 − 0.419 = 0.001（几乎没下降，分得很烂）

(b) 按 Annual Income <= 50k 分

  • ≤ 50k 一组：ID 1, 2, 9, 10 → 4 人，其中 Fraud=YES：ID 1, 9 → 2 YES, 2 NO
    Gini = 1 − (2/4)² − (2/4)² = 1 − 0.25 − 0.25 = 0.5
  • > 50k 一组：ID 3, 4, 5, 6, 7, 8 → 6 人，其中 Fraud=YES：ID 7 → 1 YES, 5 NO
    Gini = 1 − (1/6)² − (5/6)² = 1 − 0.0278 − 0.6944 = 0.278
  • 加权 Gini = (4/10)(0.5) + (6/10)(0.278) = 0.2 + 0.167 = 0.367
  • ΔGini(b) = 0.42 − 0.367 = 0.053
(c) 结论：ΔGini(b) = 0.053 > ΔGini(a) = 0.001。
所以 Annual Income ≤ 50k 这个分裂条件让节点变得更纯（Gini 下降更多），应当作为分裂属性。

"Technique: SMOTE (Synthetic Minority Over-sampling Technique). SMOTE generates new synthetic minority samples by linearly interpolating between an existing minority point and one of its k nearest minority neighbors.

Q1.3 Benford + 卡方检验（10 分）

题目：给观察频率（O%），算期望频率（E%）+ 卡方值，判断是否欺诈。

(a) Benford 在金融报表欺诈中的应用步骤：

"Benford's Law states that in naturally occurring numerical datasets, the first-digit distribution follows P(d) = log₁₀(1 + 1/d). The application steps are:
① Extract first digits of all transaction amounts in the suspect dataset.
② Count observed frequencies for each digit 1-9, calculate observed % (Oi).
③ Compute expected frequencies (Ei) using the Benford formula × n.
④ Calculate chi-square statistic: χ² = Σ(Oi − Ei)² / Ei.
⑤ Compare with critical value 15.51 (df=8, p=0.05). If χ² > 15.51, reject H₀ → data deviates from Benford → red flag.
⑥ Conclusion: Deviation alone is NOT proof of fraud — initiate further investigation (document review, interviews, transaction tracing)."

② Data Quality Issue: upstream pipeline broke, a feature is silently null, or a new merchant category code wasn't seen in training.
Diagnosis: monitor each feature's null rate, distinct-value count, range, and outlier rate vs training baseline. Run a daily 'data health' dashboard.
③ Adversarial Behavior: fraudsters discovered the model's threshold and craft transactions that score just below the cut-off (e.g., $999.99 instead of $1,000).
Diagnosis: examine the score distribution of recently-missed fraud cases. If clustered just below the threshold, that's adversarial evasion. Run adversarial robustness testing (HopSkipJump / Boundary Attack simulations).
Mitigation: schedule periodic retraining (monthly or when SSI ≥ 0.25), add online learning components, deploy A/B test new models, keep human-in-the-loop review for high-amount alerts."

题目：10 个客户 (Annual Income vs Fraud Yes/No)。用分裂条件 "Annual Income < 120k" 算 Information Gain (基于 Entropy)。

统计：10 笔，4 Yes / 6 No。

Step 1: 父节点 Entropy
Entropy(parent) = −(4/10)·log₂(0.4) − (6/10)·log₂(0.6)
= −0.4 × (−1.3219) − 0.6 × (−0.7370)
= 0.5288 + 0.4422 = 0.9710 bits

Step 2: 按 Annual Income < 120k 分

  • 左堆（< 120k）：ID 1, 2, 6, 7（90k, 100k, 110k, 105k）→ 4 人，1 Yes / 3 No
    Entropy(left) = −(1/4)·log₂(0.25) − (3/4)·log₂(0.75)
    = 0.25 × 2 + 0.75 × 0.4150 = 0.5 + 0.3113 = 0.8113 bits
  • 右堆（≥ 120k）：ID 3, 4, 5, 8, 9, 10 → 6 人，3 Yes / 3 No
    Entropy(right) = −(3/6)·log₂(0.5) − (3/6)·log₂(0.5)
    = 0.5 × 1 + 0.5 × 1 = 1.0000 bit

Step 3: 加权平均 Entropy
= (4/10)(0.8113) + (6/10)(1.0000)
= 0.3245 + 0.6000 = 0.9245 bits

Step 4: Information Gain
IG = Entropy(parent) − Weighted Entropy(children) = 0.9710 − 0.9245 = 0.0465 bits

"Information Gain = 0.0465 bits. The split 'Annual Income < 120k' yields a small reduction in entropy — meaning income alone is a weak predictor of fraud in this dataset. The right child remains 50/50 (entropy = 1.0), so this feature poorly separates fraud from non-fraud. Consider trying a different splitting variable or threshold (e.g., income at a different cut-off, or a multi-variable rule)."

The 3 false alarms can be filtered through a manual review tier; the cost is acceptable given perfect fraud capture."

Q2.2 业务解读 — 4 个问题（A+ 答法）

"(a) Who can access information most efficiently?
A and F — both have the highest Closeness centrality (0.778). They sit closest on average to all other nodes, so any information / rumor / fraud signal flowing through the network reaches them first. They are 'best informed'.

(b) Who controls information flow (broker / bridge)?
A — has the highest Betweenness (0.3968), followed by E (0.3254). A is the principal bridge through which most shortest paths between other employees pass. In fraud terms, this is the 'money mule' position: cutting A would fragment the network significantly. Strong control over information flow → potential intermediary in collusion.

(c) Who tends to imitate neighbors' behavior?
The node with the highest Relational Neighbor (P(F|?)), i.e., the highest fraction of fraudster neighbors. Per the principle of Homophily / Guilt-by-association, nodes most exposed to fraudster contacts are most likely to adopt similar behavior. (Compute from the given fraud labels.)

(d) Who should be monitored for potential fraud?
A and F — both have the highest PageRank scores (0.2042, 0.2009) and high Closeness/Betweenness. A high PageRank when seeded by known-fraud nodes indicates the node receives the strongest 'fraud signal propagation' through the network. Monitor A first: it combines high PageRank, highest Betweenness (control), and high Closeness (information access) — the 'apex risk' profile."

⚠️ SHEET F — 10 大易错提醒（考前最后看一眼）

  1. PEP 已不再仅指境外（2023 AMLO 修订纳入 domestic PEP）。
  2. 欺诈三角顺序：Pressure → Opportunity → Rationalization（Pressure 在前）。
  3. Z-score 异常值：用 |Z| > 3 而不是 Z > 3。
  4. CDD 是 5 条触发（含电汇 ≥ HK$8,000），不是 4 条。
  5. Benford 违背 ≠ 欺诈证据，仅是红旗。
  6. 不平衡数据：ROC-AUC 可能误导，优先看 PR-AUC。
  7. Transferability：集成模型不一定能防对抗攻击 — LR ↔ RF 迁移成功率 94%（"Ensemble Vulnerability Paradox"）。
  8. Chain of Custody 缺失 → 证据法庭不可采 (inadmissible)。
  9. Tipping-off 是独立罪行 — 别在题里说"告诉客户已申报 STR"。
  10. Forensic Accounting ≠ Audit：前者全样本/法律导向（Fact finding），后者抽样/财报导向（Reasonable Assurance）。
  11. Brier Score 方向：越小越好（MSE 度量），不是越大越好。
  12. Poisoning 3 类别忌混淆：Indiscriminate（让模型整体废）/ Targeted（特定类型失灵）/ Backdoor（隐藏触发器，结尾 ".37" 这类）。
  13. AML 三大挑战数字：假阳率 90%、年损失 $12.9M、隐私三件套 = Federated Learning / MPC / Differential Privacy。
